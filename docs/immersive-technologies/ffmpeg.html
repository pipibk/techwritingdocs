FFmpeg for Immersive Media Processing

Estimated reading time: 25 minutes | Skill level: Intermediate/Advanced | Written by: Rachael O. Afolabi, 2025

Overview

This guide covers a hands-on introduction to FFmpeg, a powerful command-line tool for processing video, audio, and immersive media. With clear examples, ffmpeg helps you convert, edit, and prepare media for web, streaming, and VR platforms.

What is FFmpeg?​

FFmpeg is an open-source multimedia framework for processing, converting, and streaming audio and video. It supports numerous codecs and formats, making it a crucial tool for immersive media applications, including 360-degree video, VR content, and spatial audio processing.

Why FFmpeg is Important in Immersive Media​

  • Supports high-resolution and multi-stream media formats essential for VR/AR.
  • Enables real-time encoding, decoding, and streaming.
  • Works across different operating systems.
  • Allows batch processing for automation.
  • Open-source and highly customizable for media pipelines.

Installing FFmpeg​

Download the latest FFmpeg build from the official site:

https://ffmpeg.org/download.html

Windows​

To install FFmpeg on your Windows device, follow these steps:

  • Extract the downloaded ZIP file.

  • Move the extracted folder to C:\ffmpeg (or your preferred location).

  • Add FFmpeg to system PATH:

  • Search for Environment Variables in Windows search.

  • Click Environment Variables, and find the Path.

  • Click Edit.

  • Add C:\ffmpeg\bin and click OK.

  • Open the command prompt to verify installation. Enter the following in the command prompt to verify installation:

ffmpeg -version

MacOS​

To install FFmpeg on your MacOS device, install Homebrew if not already installed:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Install FFmpeg via Homebrew:

brew install ffmpeg

Verify installation:

ffmpeg -version

Linux (Ubuntu/Debian-based systems)​

To install FFmpeg on your Ubuntu system, run the following commands in your terminal. These will update your package list, install FFmpeg, and verify the installation:

sudo apt update
sudo apt install ffmpeg -y
ffmpeg -version

Once complete, you will see the installed FFmpeg version displayed, confirming a successful setup.

Basic FFmpeg Commands​

Check Media File Information​

ffmpeg -i input.mp4
Explanation and Use Case

Purpose: Displays detailed information about the media file (e.g., codecs, duration, streams).

-i input.mp4: Specifies the input file. -i stands for "input".

Output: Printed to terminal; includes metadata like video/audio codecs, resolution, duration, and stream info.

Use Case: View details like duration, codec, bitrate, resolution, etc.

Convert a Video File​

ffmpeg -i input.mp4 output.avi
Explanation and Use Case

Purpose: Converts a video from one format to another.

input.mp4: The source video file.

output.avi: The target file in a new format (AVI in this case).

FFmpeg auto-selects default encoders based on the file extension.

Use Case: Convert a video from one format to another. For example, this converts a vacation video from .mov to .avi (your file will be whatever you have named it.)

ffmpeg -i vacation.mov to vacation.avi

Extract Audio from a Video​

ffmpeg -i input.mp4 -q:a 0 -map a output.mp3
Explanation and Use Case

Purpose: Extracts the audio track from a video and saves it as an MP3.

-q:a 0: Sets audio quality (lower number = higher quality).

-map a: Selects only the audio stream (a = audio).

output.mp3: The resulting audio file.

Use Case: Save the audio track as a separate file. This extracts high-quality audio from tutorial.mp4 and saves it as tutorial_audio.mp3.

ffmpeg -i tutorial.mp4 -q:a 0 -map a tutorial_audio.mp3

Resize a Video​

ffmpeg -i input.mp4 -vf scale=1280:720 output.mp4
Explanation and Use Case

Purpose: Changes the resolution of the video.

-vf scale=1280:720: Applies a video filter (-vf) to resize the video to 1280×720 pixels.

output.mp4: The resized video file.

Use Case: Reduce or increase video resolution. This resizes full_hd.mp4 to a 640×360 resolution and saves it as resized_sd.mp4.

ffmpeg -i full_hd.mp4 -vf scale=640:360 resized_sd.mp4

Trim a Video​

(Start at 00:00:10, duration 5 seconds)
ffmpeg -i input.mp4 -ss 00:00:10 -t 5 -c copy output.mp4
Explanation and Use Case

Purpose: Extracts a clip starting at 10 seconds and lasting 5 seconds.

-ss 00:00:10: Start time (10 seconds).

-t 5: Duration (5 seconds).

-c copy: Copies streams without re-encoding (faster, no quality loss).

output.mp4: The trimmed video file.

Use Case: Extract a short clip from a video. This trims a 10-second clip starting at 1 min 20 sec from interview.mp4 and saves it as highlight.mp4.

ffmpeg -i interview.mp4 -ss 00:01:20 -t 10 -c copy highlight.mp4

Change Frame Rate​

ffmpeg -i input.mp4 -r 30 output.mp4
Explanation and Use Case

Purpose: Changes the frame rate of the video to 30 frames per second (fps).

-r 30: Sets output frame rate.

output.mp4: The resulting file with the new frame rate.

Use Case: Adjust video smoothness by modifying frame rate. This converts animation.mov to a 60 fps video named animation_60fps.mp4.

ffmpeg -i animation.mov -r 60 animation_60fps.mp4

Advanced FFmpeg for Immersive Media​

Converting 360-Degree Video (Equirectangular to Flat)​

ffmpeg -i input.mp4 -vf v360=equirect:flat output.mp4
Explanation

This generates a regular flat video preview_clip.mp4 from the 360 video immersive_360.mp4.

-vf v360=equirect:flat: Converts from an equirectangular 360 projection to a regular flat frame.

Converting a Video to VR-Ready Format​

ffmpeg -i input.mp4 -vf v360=equirect output_vr.mp4
Explanation

This converts a standard video into equirectangular projection for use in VR platforms or players.

-vf v360=equirect: Wraps a flat video into an equirectangular projection format (needed for VR playback).

Encoding Video for WebVR Streaming​

ffmpeg -i input.mp4 -c:v libx264 -preset slow -crf 22 -b:v 3M -c:a aac -b:a 128k -movflags +faststart output.mp4

This creates vr_stream.mp4 optimized for WebVR platforms and prepares 360 or VR videos for smooth online playback in WebVR/WebXR applications.

Explanation

-c:v libx264: Use H.264 encoding (widely supported in browsers).

-preset slow: Better compression efficiency.

-crf 22: Sets visual quality (lower = better).

-b:v 3M: Sets target video bitrate.

-c:a aac: Encodes audio using AAC (standard for web).

-b:a 128k: Audio bitrate.

-movflags +faststart: Optimizes for progressive download (starts playing before full load).

Spatial Audio Processing​

ffmpeg -i input_audio.wav -af "aformat=channel_layouts=5.1" output_audio.wav
Explanation

This transforms stereo_mix.wav into a 5.1

This prepares audio for spatial (directional) surround file suitable in immersive (VR/AR scenes)environments.

-af "aformat=channel_layouts=5.1": Converts or formats audio into 5.1 surround sound layout.

Adding Metadata for 360-Degree Video​

python spatialmedia -i input.mp4 output_with_metadata.mp4
Explanation

This command uses the Spatial Media Metadata Injector, a Python tool, to inject metadata required by platforms to enable 360-degree viewing. For example, tag a video as 360° so players (e.g., YouTube, Oculus, VR headsets) recognize it properly.

Automation with FFmpeg​

Batch Convert All Videos in a Folder​

for file in *.mov; do ffmpeg -i "$file" "${file%.mov}.mp4"; done

Compress Videos for Web​

ffmpeg -i input.mp4 -vcodec libx265 -crf 28 output_compressed.

Troubleshooting and Best Practices​

Common Errors and Fixes​

  • ffmpeg: command not found → Ensure FFmpeg is installed and added to the system path.
  • Output file too large → Use lower bitrates or apply compression techniques.
  • Audio/video out of sync → Use -async 1 or -vsync vfr.

Best Practices for Optimal Performance​

  • Always check file compatibility before conversion.
  • Optimize compression settings for balance between quality and file size.
